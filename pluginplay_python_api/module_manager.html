ModuleManager Class

class pluginplay.ModuleManager

Manages the Module instances known to PluginPlay.

The goal of the ModuleManager is to make it easier for users to interact with the call graph. To this end, each node is assigned a unique label called its “module key”. Instead of specifiying a path through the call graph many operations can then be done by referring to a node by its key.

__init__()

Creates a new, empty ModuleManager.

For the purposes of this mock-up each ModuleManager is just a thin wrapper around a dictionary. The real ModuleManager is also charged with managing some additional runtime aspects, which we ignore.

add_module(key, mod)

Registers a Module instance under the provided key.

The ModuleManager is basically a dictionary from module keys to modules. This function registers a new module with the ModuleManager.

Parameters

  • key (str) – The name to store the module under.

  • mod (Module) – The module we are registering with this ModuleManager.

Raises

KeyError – If key is already in use.

copy_module(old_key, new_key)

Deep copies the specified module.

If two different callback points are set to the same module key they will alias eachother (chaning the inputs or submods for one, will also change them for the other). Sometimes we don’t want that. This function will create a deep copy of a module breaking the aliasing. The resulting module will be unlocked, allowing the user to change the inputs.

Parameters

  • old_key (str) – The key of the module being deep copied.

  • new_key (str) – The key to store the copy under.

Raises

  • KeyError – If there is no module under old_key

  • KeyError – If there is already a module under new_key

erase(key)

Removes the specified module from the ModuleManager.

After calling this function no module will be stored under key. Note that if key is assigned as a submodule already this call will not unassign it.

Parameters

key (str) – The key of the module being removed.

rename_module(old_key, new_key)

Changes the key of a module.

This function can be used to store a Module under a different key. After this call the Module will no longer be stored under the original key, leaving it free for use again.

Parameters

  • old_key (str) – The key we are moving from.

  • new_key (str) – The key we are moving to.

Raises

  • KeyError – If there is no Module under old_key

  • KeyError – If there is already a Module under new_key.

change_input(mod_key, opt_key, value)

Wraps the process of changing a Module’s input value.

This function makes it easier to change the input of a module directly through the ModuleManager. It ultimately just wraps getting the module and then calling change_input on the Module.

Parameters

  • mod_key (str) – The key of the Module whose input is being changed.

  • opt_key (str) – The key of the option whose value is being changed.

  • value – What the input should be changed to.

Raises

  • KeyError – If there is no Module under mod_key.

  • KeyError – If there is no input under opt_key.
